# **ReviewLens: Multi-Agent Customer Review Intelligence System**

This document summarizes the architectural blueprint and implementation strategy for building an AI-powered customer review intelligence system, supporting both a guest-facing review page (B2C) and a property owner dashboard (B2B).

## **1\. Core Capabilities**

* **Abstractive Summarization:** Generates cohesive, human-like paragraphs summarizing overall sentiment.  
* **Aspect-Based Sentiment Analysis (ABSA):** Evaluates specific categories (Location, Cleanliness, Service) and assigns sentiment descriptors.  
* **Keyphrase Extraction & Semantic Search:** Pulls out frequently occurring mentions and maps them to semantic vectors to allow users to filter reviews by meaning, not just exact keywords.

## **2\. Ingestion Strategy & Automation (FastAPI \+ LangGraph)**

The system uses a stateful orchestration approach using LangGraph to process reviews identically across two paths:

* **Initial Bulk Migration (e.g., 50k reviews):** A worker script processes unprocessed reviews in batches concurrently, utilizing an is\_processed flag in MongoDB for resumability.  
* **Incremental Live Updates:** A FastAPI endpoint (POST /reviews/ingest) receives single new/edited reviews and pushes them through the same LangGraph workflow for near real-time updates.

## **3\. The 6-Stage Multi-Agent Pipeline**

The core intelligence engine uses a 6-stage conditional workflow, balancing fast local LLMs (like Ollama Llama 3.1) and powerful cloud LLMs (like GPT-4o).

1. **Preprocessing Agent:** Cleans text, handles emojis, and detects language.  
2. **Deduplication Agent:** Performs exact hash and fuzzy deduplication, and scores bot probability. High bot scores short-circuit the pipeline to save costs.  
3. **Sentiment Analysis Agent:** Extracts feature-level sentiment and detects sarcasm.  
4. **Trend Detection Agent:** Classifies issues as systemic vs. isolated incidents.  
5. **Recommendations Agent (Cloud):** Generates actionable work items for property owners.  
6. **Report Synthesis Agent (Cloud):** Compiles a unified report object for storage.

## **4\. Database Architecture (MongoDB Atlas)**

All data is stored in MongoDB Atlas documents. The mentions are aggregated for the UI tags, while the embedding is used for Atlas Vector Search.  
{  
  "\_id": "rev\_10293",  
  "hotel\_id": "hotel\_abc",  
  "text": "The location is perfect, just a few blocks from Times Square.",  
  "rating": 4,  
  "mentions": \["perfect location", "times square", "corner room"\],   
  "embedding": \[0.0152, \-0.0241, 0.0881, ...\],  
  "is\_processed": true  
}

## **5\. Dual-Frontend Service Delivery**

* **B2C (Guest Application):** Consumes lightweight data. Uses GET /popular-mentions (fast aggregation) and GET /reviews/search (Atlas Vector Search).  
* **B2B (Property Owner Dashboard):** Consumes heavyweight analytical reports, surfacing systemic issues and actionable maintenance/coaching tickets generated by the advanced LangGraph agents.
